## 2. 滤波器参数

- 滤波器阶数：512（抽头数513）
- 使用ARM CMSIS-DSP库的FIR滤波器实现
- X/Y轴共享系数，但状态缓冲区独立

## 3. 低通滤波器配置

### 3.1 默认配置

系统默认使用`FIR_LOW_000_035`（0.35Hz低通滤波器）作为X/Y轴加速度计的初始滤波器：

```c
FIR_FilterType target_filter = FIR_LOW_000_035; // 默认0.35Hz低通
static FIR_FilterType current_xy_filter = FIR_LOW_000_035;
```

### 3.2 低通滤波器应用场景

低通滤波器主要用于以下场景：
- 静止状态（旋转频率接近0Hz）
- 极低速运动（旋转频率<0.2Hz）
- 需要保留静态和准静态信号的场景

## 4. 滤波器切换逻辑

### 4.1 切换触发条件

滤波器切换基于旋转频率（`rotation_freq`）进行判断：

```c
float rotation_freq = fabs(filtered_sensor_signal.gz_dps / 360.0f);
```

旋转频率 = |陀螺仪Z轴角速度| / 360°

### 4.2 切换阈值

当前切换逻辑如下：
1.低通滤波
当rotation_freq <0.2 Hz 目标滤波器是低通滤波器FIR_LOW_000_035
2.带通滤波
当rotation_freq >=0.2 Hz  目标滤波器是带通滤波器中距离中心频率最近的滤波器
中心频率为带通滤波器两个通带之和的1/2

### 4.3 迟滞机制

当target_filter与current_filter不一致时，为防止频繁切换，系统实现了迟滞机制：
1.低通和带通之间的迟滞
1.1 低通->带通
当rotation_freq>=0.2Hz时切换到带通
当rotation_freq<=0.1Hz时从带通切换到低通

（2）带通与带通之间的迟滞
当rotation_freq与当前滤波器的中心频率差的绝对值>0.2Hz时才切换


迟滞机制确保：
- 当从低通向带通切换时，只有当频率大于0.7Hz才会切换
- 当从带通向低通切换时，只有当频率小于0.6Hz才会切换

## 5. 滤波器切换实现

滤波器切换通过`fir_acc_xy_switch_type`函数实现：

```c
// 动态切换X/Y轴滤波器类型（不进行状态重置）
void fir_acc_xy_switch_type(FIR_FilterType new_type) {
    // 复制新系数到共享RAM，不执行init函数避免状态缓冲区归零
    memcpy(fir_acc_xy_shared_coeffs, fir_coeffs_flash[new_type], sizeof(float32_t)*FILTER_TAP_NUM);
    // 保留状态缓冲区，仅替换系数，无需初始化滤波器实例
}
```

切换时只更新滤波器系数，保留状态缓冲区，避免信号突变。

## 6. 滤波器应用

在信号处理流程中，滤波器应用如下：

```c
// 5. 对x，y轴加速度计进行滤波
arm_fir_f32(&fir_acc_x_instance, &sensor_signal.ax_raw, &filtered_sensor_signal.ax_g, 1);
arm_fir_f32(&fir_acc_y_instance, &sensor_signal.ay_raw, &filtered_sensor_signal.ay_g, 1);
```

## 7. 设计优点

1. **动态适应性**：根据运动状态动态调整滤波特性
2. **迟滞机制**：通过迟滞机制避免频繁切换
3. **状态保持**：切换时保留状态缓冲区，避免信号突变
4. **频率优化**：针对不同频率范围使用最优的滤波器类型

## 8. 注意事项

1. 当前配置主要针对低频运动（<3.7Hz）进行了优化
2. 如果应用场景中可能出现更高频率的运动，可能需要考虑扩展滤波器类型或调整现有参数
3. 滤波器切换可能会引入短暂的信号波动，迟滞机制可以减轻这种影响
